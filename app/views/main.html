Cette application web liste les cinémas français actifs pour l'année 2013 ainsi que les départements français.

Les opérations de CRUD standard sont disponibles pour les deux entités (cinémas et départements) de l'application.

Les données proviennent du site d'open-data de l'état français.

Technologies utilisées

Frontend

Yeoman - Web scaffolding tool

Modern workflow for modern webapps

Google, 2012 - BSD

Yeoman est un workflow pour le développement d'applications web au travers d'un generator.

Il est bâti sur npm et git et inclut un outil en ligne de commande, yo, qui permet principalement d'intéragir avec les generators et utilise grunt et bower.

En générant une application web via yo, ce dernier installe et configure l'ensemble des outils et dépendances nécessaires au projet en fonction du generator choisi, permettant au développeur de se concentrer sur son coeur de métier.

Grunt - JavaScript task runner

The JavaScript Task Runner

Bocoup, 2012 ? - MIT

L'équivalent javascript de ant dans le monde java.

grunt permet d'automatiser des opérations rébarbatives comme la minification du code, les builds, le lancement des tests, le lancement d'un serveur web de développement, le contrôle de qualité du code, le déploiement, etc ...

De nombreuses tâches développées par la communauté sont disponibles.

Bower - Web package manager

A package manager for the web

Twitter, 2012 ? - MIT

Bower est un gestionnaire de paquets. Il télécharge le code source des paquets directement sur GitHub et installe chaque paquet dans le répertoire du projet.

Une interface de recherche permet de rechercher parmi les paquets disponible.

Bower bénéficie d'une énorme popularité, il est rare de ne pas trouver un artefact via Bower.

Bootstrap - HTML/CSS framework

Sleek, intuitive, and powerful mobile first front-end framework for faster and easier web development

Twitter, 2011 - MIT

Bootstrap est un framework HTML/CSS.

C'est le projet le plus populaire sur GitHub.

Bootstrap augmente considérablement la productivité des développeurs frontend en fournissant une structure HTML générale à un projet web ainsi qu'une multitude de widgets et classes CSS.

Depuis la version 2, le framework s'est recentré sur le responsive design permettant au développeur d'avoir un code unique compatible mobile/tablette/desktop.

Angular - JavaScript framework

HTML enhanced for web apps!

Google, 2009 - MIT

AngularJS est un framework MVC javascript.

C'est un framework structurant pour single-page application, il fournit entre autre un système de binding entre le DOM et le modèle ainsi qu'un système de directive permettant d'enrichir le HTML.

Angular utilise intensivement l'injection de dépendances et facilite le TDD.

Combiné à Bootstrap via Angular-UI, il permet au développement web d'arriver à hauteur des clients lourds tout en conservant une grande légèreté.

Backend

MySQL - Relational database

The world's most popular open source database

Oracle, 1995 - GPL / Commerciale

MySQL est un système de gestion de base de données relationelles que l'on ne présente plus.

Disponible sur toutes les plateformes usuelles, il a été popularisé en son temps sur la stack LAMP.

 Play! - Java/Scala framework

The High Velocity Web Framework For Java and Scala

Zenexity, 2007 - Apache 2

Play! est un framework scala et java pour le développement d'applications web.

Il a la particularité de ne pas être basé sur l'api servlet de java.

Play! se veut être :

  • stateless (et donc scalable facilement)
  • RESTful
  • asynchrone
  • basé sur une architecture MVC

Dans le cadre de ce projet, je n'utilise pas du tout la partie vue qui est déléguée au frontend. Le backend expose une API REST (via les contrôleurs Play et les routes) avec laquelle le frontend dialogue.

L'architecture n'est donc plus réellement MVC mais est devenue SOA via l'intégration de Guice.

Play! s'utilise au travers d'un CLI pour générer l'application, lancer un serveur de développement, exécuter les tests, gérer les dépendances, etc ...

Guice - Dependency Injection

Guice (pronounced 'juice') is a lightweight dependency injection framework for Java 5 and above, brought to you by Google.

Google, 2008 - Apache 2

Guice est un framework d'injection de dépendances pour applications java.

Associé à Play! il permet de bénéficier des avantages de ce design pattern sans avoir besoin d'intégrer un composant beaucoup plus lourd comme Spring.

Outils divers

Git - Distributed revision control & SCM

--fast-version-control

Linus Torvalds, 2005 - GPL v2

git est un gestionnaire de code source créé à l'origine par Linus Torvalds pour gérer les développements sur le noyau linux.

Il a depuis été largement diffusé et est devenu une référence en la matière.

Il bénéficie d'une architecture distribuée particulièrement adaptée aux développement opensource. Sa fiabilité le rend également très populaire pour des développements en entreprise plus classiques.

IntelliJ IDEA - IDE

Work Miracles in Java and Beyond

JetBrains, 2001 - Apache 2 / Propriétaire

IntelliJ IDEA est un EDI java développé par la société tchèque JetBrains.

IntelliJ se décline en deux version, la community et la ultimate. La première est opensource et gratuite, la deuxième est propriétaire et payante.

IntelliJ se démarque des autres EDI java par sa légèreté et ses capacités de gestion du code source. Il intégre parfaitement de très nombreux frameworks et outils.

Heroku - PaaS

Deliver apps, the right way.

Salesforce.com, 2007

Heroku est une Platform as a Service (PaaS).

Elle fournit au développeur un moyen simple et efficace de déployer et gérer des applications de type Software as a Service (SaaS).

En respectant les 12 facteurs, le développeur peut bénéficier des avantages de la plateforme, scalabilité en quelques clics, de nombreux services (mails, base de données, logs, benchs, ...), ...

Le développeur doit installer la heroku toolbelt sur la machine servant au déploiement. Il pousse la branche git à déployer directement sur heroku (traité alors comme un simple remote), lequel construit et déploie tout seul le projet en fonction d'un fichier de configuration. L'application peut alors être gérée depuis la toolbelt et le dashboard fourni sur heroku.com.
